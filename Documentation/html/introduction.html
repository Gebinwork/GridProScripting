Welcome to GridPro's Python API documentation!

This is a python implementation of GridPro's Topology Input Language(TIL).The aim of making this accessible to the user is to provide greater flexibility in using the software for automation purposes. The topology input language (TIL) is used to record the topology into files. The topology files must have the file name extension ‘.fra’. The design and recording process can be done either by manually programming in TIL or using WS GUI. While programming the TIL file the user specifies mainly a sequence of corner definitions, each of which provides an approximate initial position of the current corner, a list of other corners that has a link to the current corner, and a list of surfaces to which the corners have to be projected. The Grid generator takes the input and creates the information about the edges, faces and blocks automatically!

With the help of this API, users will have the flexibility to define the building process of a topology file from scratch without any GUI access. This will facilitate in automation of blocking for geometries in a given design space. Alternatively the API can also be used to manipulate existing topology files to suit the users need and perform automation tasks.

Classes:

Following are the major classes and object types of the API. All the relevant member functions are grouped under it's respective object type. Please note that all of these object types come under the class called TOPOLOGY except vector3D, which by itself is a class.

Corner

Corner data structure stores corner information like positioning, corner links and their assignment details

Surface

A topology cannot be defined without a set of bounding surfaces. Surface data structure stores the information common to all kinds of surfaces. It stores the surface index, transformation details, orientation, clustering parameters, label and property of the surface. There are various types of surfaces that can be loaded into GridPro

Topology

Topology data structure containing corners, surfaces and other information. It functions as both class and object type.

Corner Group

Corner groups helps you manipulate 3D topology effectively.

Surface Group

Surface group is as much useful as corner groups as it helps you to differentiate groups of surfaces from all the surfaces in the topology.

Singularities

Singularity usually means singular edges in 3D and singular points in 2D. There are primarily used to steer the grid lines to better fit a complex geometry. GridPro restricts the degree of a singularity to be either 3, or 5 to 6.

Density

Density is a number that describes the number of grid cells along an edge (and thus all compatible edges)

Vector3D

Constructor which accepts 3 coordinates. It should be used only in 3D Topologies.

Example Usage:

		# IMPORT OPERATIONS
		import gp_utilities
		from gp_utilities import vector3d as vec
		
		#Main Function
		if(__name__ == '__main__'):
		topo = gp_utilities.Topology()      #Assign Topology Class
		
		#Creating surface for each face of the cube
		s0 = topo.add_plane(vec(0, 0.5, 0.5), vec(-0.4,0,0))
		s1 = topo.add_plane(vec(1, 0.5, 0.5), vec( 0.4,0,0))
		s2 = topo.add_plane(vec(0.5, 0, 0.5), vec(0,-0.4,0))
		s3 = topo.add_plane(vec(0.5, 1, 0.5), vec(0, 0.4,0))
		s4 = topo.add_plane(vec(0.5, 0.5, 0), vec(0,0,-0.4))
		s5 = topo.add_plane(vec(0.5, 0.5, 1), vec(0,0, 0.4))
		
		#Corner Creation
		c0 = topo.add_corner(0, 0, 0)     #Corner id 0
		c1 = topo.add_corner(1, 0, 0)     #Corner id 1
		c2 = topo.add_corner(0, 1, 0)     #Corner id 2
		c3 = topo.add_corner(1, 1, 0)     #Corner id 3
		c4 = topo.add_corner(0, 0, 1)     #Corner id 4
		c5 = topo.add_corner(1, 0, 1)     #Corner id 5
		c6 = topo.add_corner(0, 1, 1)     #Corner id 6
		c7 = topo.add_corner(1, 1, 1)     #Corner id 7

		#Linking the corners
		topo.link([(c0,c1), (c0,c2), (c2,c3), (c1,c3),
				   (c4,c5), (c4,c6), (c6,c7), (c5,c7),
				   (c0,c4), (c1,c5), (c2,c6), (c3,c7)])

		#Assigning the corners to each of the surface
		c0.assign(s4)
		c0.assign(s2)    #c0.assign_multiple([s4,s2,s0])
		c0.assign(s0)
		
		c1.assign(s4)
		c1.assign(s2)
		c1.assign(s1)
		
		c2.assign(s4)
		c2.assign(s3)
		c2.assign(s0)

		c3.assign(s4)
		c3.assign(s3)
		c3.assign(s1)

		c4.assign_multiple([s5,s2,s0])

		c5.assign_multiple([s5,s2,s1])
		
		c6.assign_multiple([s5,s3,s0])

		c7.assign_multiple([s5,s3,s1])

		#Writing the fra file
		topo.write_topology("cube.fra")
