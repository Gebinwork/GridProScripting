Surface:

A topology cannot be defined without a set of bounding surfaces. Surface data structure stores the information common to all kinds of surfaces. It stores the surface index, transformation details, orientation, clustering parameters, label and property of the surface. There are various types of surfaces that can be loaded into GridPro.

The syntax for this object type is: surface( )

Every object type comes under the class called TOPOLOGY. And it has two sets of member functions, One set is the list of functions which execute the defined operations and the other set is the list of functions which gets the necessary information. Following are the list of member functions under this object type:

Operations:
  • delete_surface
  • set_first_cell_height
  • set_stretch
  • set_load_only_once
  • set_ewall_spacing
  • set_label
  • set_pty
  • set_orientation
  • label_faces
Query Functions:

  • get_load_only_once_status
  • get_property
  • get_first_cell_height
