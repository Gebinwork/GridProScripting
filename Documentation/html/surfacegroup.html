Group:

One tool that helps you create three-dimensional topologies is the workplane; another tool is group. Often, you will repeat procedures for creating topologies because grid topology often requires the user to repeat operations. GridPro has developed the group feature in order to help you circumvent the procedural repetition so prevalent in the field of grid generation.

More than any other feature, groups help you manipulate three-dimensional topology effectively. Creating a typical topology composed of hundreds of corners and edges without utilizing groups is ill-conceived, time-consuming, and unnecessary. Similar to topology groups, you can group surfaces also. Surface group is as much useful as corner groups as it helps you to differentiate groups of surfaces from all the surfaces loaded into the file.

Surface Group:
The syntax for this object type is: surface_group(group_id)

Every object type comes under the class called TOPOLOGY. And it has two sets of member functions, One set is the list of functions which execute the defined operations and the other set is the list of functions which gets the necessary information. Following are the list of member functions under this object type:

Operations:

  • add
  • add_multiple
  • add_all
  • add_all_after_index
  • remove
  • remove_multiple
  • clear
  • delete_all
  • replicate
Query Functions:
  • contains
  • size
  • get_all
