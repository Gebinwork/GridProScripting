Corner:

Corner data structure stores corner information like positioning, corner links and their assignment details.A collection of corners and their links describe the wireframe structure. So a corner (or node/point) is a basic entity of a wireframe. A corner is defined by it's position and the links joining them. For a valid topology definition, the corners should also contain the surface assignment information. Apart from this, in a periodic topology, corner stores the paired corner information for periodic definition.

The syntax for this object type is: corner(corner_id)

Every object type comes under the class called TOPOLOGY. The corner object type takes an id as the input parameter. And it has two sets of member functions, One set is the list of functions which execute the defined operations and the other set is the list of functions which gets the necessary information. Following are the list of member functions under this object type:

Operations:
  • set_coordinates
  • delete_corner
  • assign
  • unassign
Query Functions:
  • get_id
  • get_coordinates
  • get_links
  • get_assignments
  • get_paired_corner
  • get_distance
